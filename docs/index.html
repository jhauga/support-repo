ts-filecomp

Compare files by size, creation date, and modification date — locally or remotely — from your terminal or code.

TypeScript CLI + API Local & Remote Zero Dependencies

Installation

Install globally to use the CLI, or as a dev dependency for programmatic usage.

Global Install

npm install -g ts-filecomp
yarn global add ts-filecomp
pnpm add -g ts-filecomp
bun add -g ts-filecomp

Dev Dependency

npm install --save-dev ts-filecomp
yarn add --dev ts-filecomp
pnpm add -D ts-filecomp
bun add -d ts-filecomp

CLI Usage

ts-filecomp <file1> <file2> [options]

Examples

# Compare two local files (same name required by default)
ts-filecomp ./v1/report.csv ./v2/report.csv

# Compare by size only
ts-filecomp ./a/data.json ./b/data.json --size

# Compare a local file with a remote file
ts-filecomp ./data.json https://cdn.example.com/data.json --remote

# Compare files with different names (requires flag)
ts-filecomp ./old-report.csv ./new-report.csv --different-names

# JSON output
ts-filecomp ./a/config.yaml ./b/config.yaml --json

# Verbose output
ts-filecomp ./a/bundle.js ./b/bundle.js --verbose

Options

FlagShortDescription
--size-sCompare file sizes only
--created-cCompare creation dates only
--modified-mCompare modification dates only
--all-aCompare all attributes (default)
--remote-rTreat one or both paths as remote URLs
--different-names-dAllow comparing files with different names
--json-jOutput results as JSON
--verbose-vVerbose output with full metadata

Same-Name Constraint

By default, ts-filecomp enforces that both files share the same file name (the basename, ignoring directory paths). This prevents accidental comparison of unrelated files.

# ✓ Same name — works
ts-filecomp ./v1/report.csv ./v2/report.csv

# ✗ Different names — blocked by default
ts-filecomp ./old-report.csv ./new-report.csv
# Error: Files have different names. Use --different-names to compare anyway.

# ✓ Override with flag
ts-filecomp ./old-report.csv ./new-report.csv --different-names

Programmatic API

Import ts-filecomp in your TypeScript project for programmatic file comparisons.

Basic Comparison

import { compareFiles } from 'ts-filecomp';

const result = await compareFiles('./v1/data.json', './v2/data.json');

console.log(result.size);     // { file1: 2048, file2: 3072, diff: 1024, larger: 'file2' }
console.log(result.created);  // { file1: '2025-01-15T...', file2: '2025-03-01T...', newer: 'file2' }
console.log(result.modified); // { file1: '2025-06-10T...', file2: '2025-06-12T...', newer: 'file2' }

Remote Comparison

import { compareFiles } from 'ts-filecomp';

const result = await compareFiles(
  './local/data.json',
  'https://cdn.example.com/data.json',
  { remote: true }
);

Different Names

import { compareFiles } from 'ts-filecomp';

const result = await compareFiles(
  './old-report.csv',
  './new-report.csv',
  { differentNames: true, compare: ['size', 'modified'] }
);

Options Interface

interface CompareOptions {
  compare?: ('size' | 'created' | 'modified')[];  // default: all three
  remote?: boolean;                                 // default: false
  differentNames?: boolean;                         // default: false
  json?: boolean;                                   // default: false
  verbose?: boolean;                                // default: false
}

interface CompareResult {
  file1: string;
  file2: string;
  sameName: boolean;
  size?: {
    file1: number;
    file2: number;
    diff: number;
    larger: 'file1' | 'file2' | 'equal';
  };
  created?: {
    file1: string;
    file2: string;
    newer: 'file1' | 'file2' | 'equal';
  };
  modified?: {
    file1: string;
    file2: string;
    newer: 'file1' | 'file2' | 'equal';
  };
}
